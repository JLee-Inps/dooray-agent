# 제품 요구사항 (PRD)

## 한 줄 정의

NHN Dooray 문서(위키·업무)를 터미널·Claude Code 에서 **자동으로 읽고 쓰는** 에이전트 친화 CLI.

## 문제

- Dooray 웹 UI 는 사람용이라 AI 에이전트가 문서를 자동으로 조회·갱신하기 어렵다.
- 기존 CLI 는 일부 쓰기 명령이 기계가 읽는 출력을 내지 않아 자동화가 중간에 끊긴다.

## 타겟

- Dooray 를 쓰는 개발팀에서 **Claude Code 로 문서 작업을 자동화**하려는 사용자.
- 스크립트·파이프라인으로 업무·위키를 다루려는 자동화 엔지니어.

## 핵심 가치

- **모든 명령이 3-모드 출력**(표/`--json`/`--quiet`) — 에이전트가 읽고, 쓰고, 결과를 확인·체이닝한다.
- **자기탐색**(`capabilities`) — 에이전트가 능력을 런타임에 발견한다.
- **구조화된 에러** — 실패도 파싱 가능해 자동화가 견고하다.

## MVP 범위

### 포함

- 인증(login/whoami/doctor/setup/config)
- 업무: 목록·조회·생성·수정·완료·워크플로·검색 + 댓글 + 첨부파일
- 위키: 페이지 목록·조회·생성·수정 + 댓글 + 첨부파일
- 프로젝트 메타(멤버·태그·마일스톤·워크플로·그룹·템플릿), 멤버 검색
- 메일(조회·발송), 피드백, 캐시

### 제외 (후속)

- 업무 템플릿 보간, parent/milestone 지정, member-group 멘션
- 댓글 첨부, `download-all` 일괄 다운로드

## 성공 지표

- 에이전트가 사람 개입 없이 "문서 읽기 → 가공 → 쓰기 → 결과 확인" 루프를 완주.
- 모든 읽기/쓰기 명령이 `--json` 으로 파싱 가능한 결과를 낸다(예외 0).
